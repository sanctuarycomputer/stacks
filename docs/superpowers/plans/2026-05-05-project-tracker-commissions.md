# Project Tracker Commissions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a back-of-house Commission feature on `ProjectTracker` that deducts amounts off the top of each invoice line before contributor payout math runs, paid out as `ContributorPayout` records via a new `Commission` blueprint role.

**Architecture:**
- New `Commission` STI model attached to `ProjectTracker` (initial subclasses: `PercentageCommission`, `PerHourCommission`).
- Per-line deduction inside `InvoiceTracker#make_contributor_payouts!` — `working_amount` is reduced before AL/PL/IC math.
- Recipient gets a `ContributorPayout` with a new `"Commission"` blueprint role; auto-accepted when commission-only.
- Downstream math (`calculate_surplus`, 70% cap) updated to use post-commission amounts.

**Tech Stack:** Rails 6 + ActiveAdmin + Minitest with mocha + `acts_as_paranoid` for soft-delete.

**Spec:** `docs/superpowers/specs/2026-05-05-project-tracker-commissions-design.md`

class PercentageCommission < Commission
  validates :rate, numericality: { less_than_or_equal_to: 1 }

  def deduction_for_line(qbo_line_item, _blueprint_line)
    (qbo_line_item["amount"].to_f * rate.to_f).round(2)
  end

  def description_line(_qbo_line_item, blueprint_line, deduction)
    hrs = blueprint_line["quantity"].to_f
    rt  = blueprint_line["unit_price"].to_f
    "- #{hrs} hrs * #{n2c(rt)} p/h * #{(rate.to_f * 100).round(2)}% = #{n2c(deduction)} (commission to #{contributor.display_name})"
  end
end

  def contributor_payouts_within_seventy_percent
    return if ActiveModel::Type::Boolean.new.cast(skip_seventy_percent_check)
    return if changes.keys == ["accepted_at"]

    cps = invoice_tracker.contributor_payouts.include?(self) ? invoice_tracker.contributor_payouts : [*invoice_tracker.contributor_payouts, self]

    commissions_total = cps.sum(&:as_commission)

    if invoice_tracker.forecast_client.is_internal?
      post_commission_total = invoice_tracker.total - commissions_total
      max_amount = post_commission_total
    else
      post_commission_total = invoice_tracker.total - commissions_total
      max_amount = post_commission_total * (1 - invoice_tracker.company_treasury_split)
    end

    contributor_pool_sum = cps.sum { |cp| cp.amount.to_f - cp.as_commission }

    if contributor_pool_sum > (max_amount + 1)
      errors.add(:base, "Contributor Payouts may not exceed #{ActionController::Base.helpers.number_to_currency(max_amount)} (#{100 * (1 - invoice_tracker.company_treasury_split)}% of post-commission invoice total).")
    end
  end

  def calculate_surplus
    return [] unless in_sync?

    qbo_inv = invoice_tracker.qbo_invoice
    return [] unless qbo_inv.present?

    project_trackers = invoice_tracker.project_trackers

    blueprint["IndividualContributor"].map do |ic|
      blueprint_metadata = ic.dig("blueprint_metadata")
      qbo_line_item = qbo_inv.line_items.find{|li| li["id"] == blueprint_metadata.dig("id")} || {}
      amount_paid = ic.dig("amount").try(:to_f) || 0
      amount_billed = qbo_line_item.dig("amount").try(:to_f) || 0

      commission_for_line = invoice_tracker.commission_total_for_line(blueprint_metadata.dig("id"))
      working_amount = amount_billed - commission_for_line

      surplus = 0
      if working_amount > 0
        profit_margin = (working_amount - amount_paid) / working_amount

## Notes

- **No new QBO bill expense account override yet.** Commission-only CPs use the recipient's studio-derived expense account via the existing `find_qbo_account!`. This is intentional per spec — switch to a dedicated "Commissions Paid" account is a follow-up.
- **No time bounds or per-forecast-project filters.** Commissions are tracker-wide and always-on. If you find yourself reaching for these, stop and update the spec first.
- **`destroy` on a `Commission`** soft-deletes via `acts_as_paranoid`. The CP entries that referenced it remain intact (they store amount + description_line, not a foreign key to the Commission row).
- **Re-running `make_contributor_payouts!`** is the canonical regen mechanism — same as today for AL/PL changes. Adding/removing/editing commissions on a tracker only takes effect when generation is re-triggered for the affected invoices.
